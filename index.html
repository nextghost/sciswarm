Sciswarm — Introduction

What if preprints could achieve the same formal impact as papers published in a journal, but without going through the traditional publishing machinery? The main difference between preprints and published scientific papers is formal validation through peer review. What preprints need to become first-class citizens of academic libraries is a formal validation process that's separate from traditional journals. Sciswarm is an online platform which enables such validation without preventing submission to any journal at the same time.

But first things first. Sciswarm is something between a social network and a massively collaborative overlay journal. The system is currently in alpha stage and I need a few volunteers to test the usefulness of the social features. This webpage is a short tour of the Sciswarm features which aren't immediately visible to unregistered users. Link to Sciswarm itself is at the end of this tour.

Sciswarm works with paper metadata so that you can post all your papers in it, including those published in paywalled journals. But you don't need to worry about entering dozens of papers into the system yourself. Sciswarm currently contains metadata of over 1.5 million preprints from arXiv and new preprints are imported every day. So if you've read any arXiv preprints or their published versions in the past month, you can simply register, claim ownership of your own preprints with a few clicks and you're ready to start testing.

Authors can respond to a review, for example to clear up any misunderstandings. The discussion is public but read-only to everyone except the reviewer and paper authors.

And the feature that should be most useful to you from the beginning is the timeline. Here you'll find new papers, reviews and recommendations from people you follow. Which of course means that you'll need to test the system together with your colleagues.

Sciswarm also has some utility features like delegating paper management to other users and simple claiming of paper authorship through identifier or name match. But those are not the primary reason why you'd want to use the system so let's end the tour here. The user interface is far from finished and there are lots of planned features, for example:

  • Paper collections
  • Improved paper search
  • RSS feeds
  • Algorithmic paper recommendations
  • Research replication tracking
  • User groups and discussions
  • Open data, etc.

Sciswarm is free software and the system should eventually become a global network of interconnected servers owned by research institutions themselves. If you'd like to help, continue to Sciswarm and send me your feedback. My e-mail address is doucha(at)swarmtech.cz
